=== PART 2: Strip events.rs to minimal usage events ===

EDIT app/src/server/telemetry/mod.rs:

This file is ~7355 lines. We need to strip it down to ~100 lines with only:
- AppOpened — when the app starts
- AppClosed — when the app closes
- AppActive — periodic active usage signal (keeps existing timer)
- ModelUsed — which AI model the user selected (model name only, no cost)
- FeatureUsed — which feature was triggered (basic feature name only, no user data)

Delete ALL OTHER event types. The file should contain just these 5 minimal event structs.

Replace the ENTIRE file content with:
```rust
// Octomus telemetry — minimal usage analytics only.
// No user-generated content, no cost/credit tracking, no PII.

use serde::{Deserialize, Serialize};

/// App was launched.
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct AppOpened {
    pub app_version: String,
    pub os: String,
}

/// App was closed. Contains session duration.
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct AppClosed {
    pub session_duration_seconds: u64,
}

/// Periodic active usage heartbeat.
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct AppActive {
    pub timestamp: i64,
}

/// User selected an AI model. Model name only, no API key or config.
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct ModelUsed {
    pub model_name: String,
    pub provider_name: String,
}

/// A feature was used. Generic feature name only.
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct FeatureUsed {
    pub feature_name: String,
}

// Re-export all for backward compatibility with mod.rs
pub use self::AppOpened as AppOpenedEvent;
pub use self::AppClosed as AppClosedEvent;
pub use self::AppActive as AppActiveEvent;
pub use self::ModelUsed as ModelUsedEvent;
pub use self::FeatureUsed as FeatureUsedEvent;
```
